# Junior front-end developer (React)

Приложение написано на React TypeScript

## Приложение должно содержать
### 1. Таблицу, которая может содержать произвольное количество полей (от 5 до 15)

- Таблица подстраивается под количество полей (не ограничивал входящие данные с сервера, если данные не пустой массив), может содержать как 1-2 поля так и более 15 
### 2. Данные в таблицу должны загружаться с сервера и использовать Infinite Loader для подгрузки новых данных

- Для более удобного просмотра подгрузки данных с сервера выставлена задержка в 1 секунду  
### 3. Стейт-менеджер (или его отсутствие, обосновать выбор или же отсутствие инструмента)

- Сторонний стейт-менеджер не использовал, т.к. не стал особо разбивать приложение на компоненты, разделил всего на 3 компонента, поэтому использовал только `useState()`
### 4. Форма создания новой записи в таблице - должно быть минимум 5 полей

- Форму создания сделал в модальном окне.
#### i. Учитывайте, что полей может быть произвольное количество

- при загрузке данных с сервера, если приходит не пустой массив данных (`http://localhost:3000/users`), то форма принимает поля в зависимости от входящих данных с сервера, за исключением `id`, т.к. генерируется автоматически
- при пустых данных с сервера (`http://localhost:3000/empty`), форму с полями создает пользователь. Выставлены ограничения на минимальное и максимальное количество полей, 5 и 15 полей. Имеет минимальную валидацию на пустую строку и на дублирование название поля. Disable кнопки реализовал в этом моменте.
#### ii. Поля имеют валидацию

- у полей минимальная валидация, только проверка на пустую строку
#### iii. При отправке должен быть соответствующий стейт (disable кнопки или другие)

- реализовал через попап уведомление
#### iv. Сервер может не принять форму и вернуть ошибки (опять же можно не делать, но описать реализацию)

- реализовал через попап уведомление 
### 5. Форма должна отправляться по api, запись добавляться в таблицу

- Запись добавляется в конец таблицы
### 6. Любая анимация (например попап с формой)

- всплывающее уведомление при успешном создании записи на сервере и при ошибке
